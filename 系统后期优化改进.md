# 系统后期优化改进方案

> 基于 Open WebUI、Lobe Chat、ChatGPT、Claude、Gemini 等行业标杆对标分析  
> 最后更新：2026-05-11

---

## 一、核心情感陪伴体验（最高优先级）

项目定位为"情感疗愈伴侣"，以下功能直接关系到核心竞争力：

### 1.1 系统提示词/人设配置

- **现状**：数据库 `chats` 表有 `system` 列，但从未暴露给用户配置
- **影响**：AI 角色无法定制，对陪伴型应用是致命缺失
- **方案**：在设置或对话页面增加"AI 人设"编辑区，允许用户自定义 AI 的身份、性格、说话风格

### 1.2 TTS 语音朗读

- **现状**：无语音输出功能
- **影响**：AI"说话"是情感陪伴的核心体验，纯文字交互缺少温度
- **方案**：集成浏览器原生 SpeechSynthesis API（免费），或接入 Ollama 兼容的 TTS 接口

### 1.3 STT 语音输入

- **现状**：仅支持键盘输入
- **影响**：打字交流对情感倾诉场景形成心理门槛
- **方案**：集成浏览器原生 SpeechRecognition API（免费，Chrome/Edge 已支持）

### 1.4 对话分支可视化

- **现状**：树形消息结构已完整实现（`showPreviousMessage`/`showNextMessage`），但 UI 未暴露
- **影响**：功能已写好但不可用——零成本可解锁
- **方案**：在助手消息旁增加 ← → 箭头按钮，切换不同分支回复

### 1.5 对话记忆/用户画像

- **现状**：每次对话互相独立，AI 不记得用户之前说过什么
- **影响**：削弱陪伴的连续性
- **方案**：在系统提示词中注入用户历史摘要（关键词、偏好、过往话题）

## 二、对标 Open WebUI 的功能差距

Open WebUI 是与本项目最对标的产品（同为 Ollama 生态），v0.9.x 版本已具备：

### 2.1 聊天全文搜索

- **现状**：侧边栏只能按标题搜索
- **方案**：API 增加 `GET /api/chats/search?q=keyword`，侧边栏支持全文检索

### 2.2 聊天导出/导入

- **现状**：`exportChats`/`addChats` 方法已实现，但无 UI 触发
- **方案**：设置页或侧边栏增加"导出全部对话 (JSON)"和"导入对话"按钮

### 2.3 对话归档/置顶

- **现状**：无组织功能
- **方案**：每条对话加 Pin 和 Archive 标记，侧边栏分区显示

### 2.4 模型管理 UI

- **现状**：仅能选择已有模型，不能 Pull/Delete
- **方案**：设置中增加模型管理面板，调用 Ollama API 拉取/删除模型，显示模型详情（大小、量化级别等）

### 2.5 键盘快捷键

- **现状**：无快捷键
- **方案**：
  - `Enter` 发送（已实现）
  - `Shift+Enter` 换行（已实现）
  - `Ctrl+N` 新建对话
  - `Ctrl+/` 搜索对话
  - `Esc` 关闭弹窗

### 2.6 国际化 i18n

- **现状**：中英文混杂（Toast 提示中文，错误信息英文）
- **方案**：统一为中文，或引入 i18n 框架支持多语言切换

### 2.7 多用户管理面板

- **现状**：有注册/登录系统，但无管理后台
- **方案**：增加管理员角色，可查看用户列表、管理模型、监控使用情况

## 三、对标 ChatGPT/Claude 的核心体验差距

### 3.1 消息编辑

- **现状**：消息发送后无法编辑
- **方案**：用户消息旁加"编辑"按钮，编辑后重新生成回复

### 3.2 消息删除

- **现状**：只能删除整个对话，不能删单条消息
- **方案**：每条消息旁加"删除"按钮，连带删除该消息及其子分支

### 3.3 打字指示器

- **现状**：流式生成时无任何视觉提示
- **方案**：AI 头像旁或消息气泡内显示跳动的三个点动画

### 3.4 流式性能指标

- **现状**：`message.info` 对象包含完整的性能数据但从未渲染
- **方案**：在助手消息末尾显示 Token/s、生成耗时（灰色小字）

### 3.5 错误重试

- **现状**：失败消息显示错误文本，无重试操作
- **方案**：失败消息旁加"重试"按钮，重新发送上一条用户消息

### 3.6 对话分享

- **现状**：无分享功能
- **方案**：生成只读分享链接（含对话内容 + token），或导出为精美图片

### 3.7 主题跟随系统

- **现状**：仅手动切换深色/浅色
- **方案**：增加"跟随系统"选项，监听 `prefers-color-scheme`

## 四、其他行业标准功能

### 4.1 RAG 知识库

- **现状**：不支持文档上传和检索
- **方案**：长期目标，可接入 Ollama Embedding API + 向量存储

### 4.2 工具调用/插件

- **现状**：无函数调用、无代码解释器、无 Web 搜索
- **方案**：长期目标，可参考 Open WebUI 的 MCP 工具生态

### 4.3 PWA 离线支持

- **现状**：无 manifest.json、无 Service Worker
- **方案**：添加 PWA 配置，支持桌面快捷方式和离线缓存

### 4.4 加载骨架屏

- **现状**：页面加载时显示空白
- **方案**：登录/注册/聊天页面添加 Skeleton 占位符

### 4.5 密码重置流程

- **现状**：无"忘记密码"功能
- **方案**：增加密码重置 API（邮箱验证码）

### 4.6 速率限制

- **现状**：API 端点无限流
- **方案**：登录接口增加失败次数限制，防止暴力破解

### 4.7 环境变量配置

- **现状**：数据库密码、JWT 密钥硬编码在源码中
- **方案**：迁移到 `.env` 文件，通过 `process.env` 读取
