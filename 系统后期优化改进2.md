# 系统后期优化改进方案（第二轮）

> 基于 Open WebUI、Lobe Chat、ChatGPT、Claude、Kimi、豆包、通义千问、文心一言等行业产品对标分析  
> 最后更新：2026-05-11

---

## 一、情感陪伴核心能力（最大缺失）

项目定位为"情感疗愈伴侣"，以下功能直接关系到核心竞争力：

### 1.1 跨会话记忆

- **行业现状**：ChatGPT Memory 记住用户偏好、背景、历史话题；Kimi 记住用户信息
- **当前问题**：每次对话 AI 都像"陌生人"，严重削弱陪伴连续性——这是陪伴类应用最致命的缺失
- **方案**：在系统提示词中注入用户画像（关键词、偏好、过往话题摘要），每次对话开始时自动加载

### 1.2 AI 人设/系统提示词配置 UI

- **行业现状**：ChatGPT Custom Instructions、Claude Projects 都可自定义 AI 风格
- **当前问题**：数据库 `chats` 表有 `system` 列，但 UI 从未暴露给用户，AI 性格不可定制
- **方案**：在设置或对话页面增加"AI 人设"编辑区，允许用户自定义 AI 的身份、性格、说话风格

### 1.3 语音输入/输出

- **行业现状**：ChatGPT 语音模式、豆包 TTS/STT，所有主流 App 标配
- **当前问题**：纯文字交互缺少温度和沉浸感
- **方案**：
  - TTS：集成浏览器原生 SpeechSynthesis API（免费，零依赖）
  - STT：集成浏览器原生 SpeechRecognition API（免费，Chrome/Edge 已支持）

### 1.4 情绪感知/追踪

- **行业现状**：Character.AI、Replika 等陪伴应用有情绪识别
- **当前问题**：无法感知用户情绪变化，回复千人一面
- **方案**：在系统提示词中定期注入对话情绪摘要，引导 AI 适应性回复

## 二、对话体验

### 2.1 消息编辑

- **现状**：已发消息无法编辑后重新生成
- **方案**：用户消息旁加"编辑"按钮，编辑后重新生成回复

### 2.2 对话分支可视化

- **现状**：树形数据结构已实现，`showPreviousMessage`/`showNextMessage` 函数已写好，但 UI 未暴露 ←/→ 按钮
- **方案**：在助手消息旁增加 ← → 箭头按钮，切换不同分支回复（零成本解锁）

### 2.3 打字指示器

- **现状**：等待 AI 回复时无动画提示，用户不知道是否在生成
- **方案**：AI 头像旁显示跳动的三个点动画或"正在输入..."文字

### 2.4 流式指标显示

- **现状**：`message.info` 有完整的 token/s、耗时数据，但从未渲染
- **方案**：在助手消息末尾显示 Token/s、生成耗时（灰色小字）

### 2.5 自动滚动优化

- **现状**：用户向上翻阅历史时，流式内容强制拉回底部
- **方案**：用户手动滚动时暂停自动滚动，流式完成后显示"滚动到底部"按钮

### 2.6 上下文长度提示

- **现状**：无法查看当前已用/总 token 数
- **方案**：输入框附近显示 token 估算计数器

## 三、内容能力

### 3.1 文件/图片上传（多模态）

- **行业对标**：ChatGPT、Claude、Kimi 均支持
- **方案**：支持粘贴/拖拽图片，转 base64 传给支持多模态的 Ollama 模型

### 3.2 网页搜索

- **行业对标**：ChatGPT Browse、Kimi 联网搜索
- **方案**：接入 Ollama 的 tool calling 或简单的 Web Search API

### 3.3 URL 阅读

- **行业对标**：Kimi 支持粘贴 URL 自动抓取内容
- **方案**：输入 URL 时自动抓取网页文本注入上下文

### 3.4 Markdown 导出

- **行业对标**：ChatGPT 可复制为 Markdown
- **方案**：消息复制按钮增加"复制为 Markdown"选项

### 3.5 代码执行

- **行业对标**：ChatGPT Code Interpreter、Claude Artifacts
- **方案**：长期目标，接入浏览器端 WebAssembly 沙箱执行

## 四、管理与组织

### 4.1 对话置顶/归档

- **方案**：每条对话加 Pin 和 Archive 标记，侧边栏分区显示（已纳入第一轮优化方案）

### 4.2 对话标签/分组

- **现状**：侧边栏仅按日期分组（今天/昨天/本周/更早），无自定义分类
- **方案**：支持用户创建标签，按标签筛选对话

### 4.3 提示词库

- **现状**：无预设提示词模板
- **方案**：内置常用提示词模板（心理疏导、情绪安抚、日常聊天等），支持用户自定义保存

### 4.4 对话分享

- **现状**：无分享链接或导出精美图片
- **方案**：生成只读分享链接或导出为精美长图/卡片

### 4.5 数据统计

- **现状**：无使用数据
- **方案**：展示对话次数、AI 调用次数、token 消耗等统计面板

## 五、平台能力

### 5.1 PWA 安装

- **现状**：已有 `manifest.json` 但无 Service Worker，无法离线使用
- **方案**：注册 Service Worker，缓存静态资源，支持离线打开

### 5.2 桌面通知

- **现状**：代码有 `Notification` API 残留但已移除
- **方案**：AI 回复完成时发送桌面通知（用户可开关）

### 5.3 键盘快捷键

- **现状**：无快捷键体系
- **方案**：
  - `Ctrl+N` 新建对话
  - `Ctrl+/` 搜索对话
  - `Ctrl+Enter` 发送
  - `Esc` 关闭弹窗/侧边栏

### 5.4 响应式优化

- **现状**：侧边栏在移动端体验差，输入框与虚拟键盘交互不畅
- **方案**：移动端全屏对话视图 + 底部 TabBar 导航

### 5.5 多语言

- **现状**：中英混杂（Toast 中文、代码注释英文、部分 UI 英文）
- **方案**：统一为中文，或引入 i18n 框架

### 5.6 主题跟随系统

- **现状**：仅手动切换深色/浅色，无 `prefers-color-scheme` 自动跟随
- **方案**：增加"跟随系统"选项，监听系统主题变化
