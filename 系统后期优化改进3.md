## 一、情感陪伴核心体验（最大差距，全未实现）

项目定位为"情感疗愈伴侣"，但以下能力全部缺失，AI 目前只是一个"没有记忆、没有声音、没有性格"的纯文字回复器。

### 1.1 AI 人设/系统提示词配置 UI

- **行业对标**：ChatGPT Custom Instructions、Claude Projects、豆包角色扮演
- **当前状态**：数据库 `chats` 表有 `system` 列，`SettingsModal` 从未暴露该配置。AI 的性格、身份、说话风格完全不可定制
- **实现成本**：低——在设置页或对话页加一个 `<textarea>` 编辑 system prompt
- **关键价值**：有了固定的人设（如"温柔知心的女性 AI 伴侣"），陪伴才有连续性

### 1.2 跨会话记忆 / 用户画像

- **行业对标**：ChatGPT Memory、Kimi 记住用户偏好
- **当前状态**：每次开新对话 AI 都是陌生人，不知道你叫什么、之前聊过什么、有什么心事
- **方案**：在 `users` 表增加 `profile` 列（JSON），记录昵称、关键词、过往话题摘要；每次对话开始时自动注入 system prompt
- **关键价值**：陪伴型应用最致命的缺失——记忆是情感的载体，没有记忆就没有关系

### 1.3 语音输入 (STT)

- **行业对标**：豆包语音输入、ChatGPT Voice Mode、通义千问语音输入
- **当前状态**：仅支持键盘输入，无语音功能
- **方案**：接入浏览器原生 `SpeechRecognition` API（Chrome/Edge 已支持，完全免费）
- **关键价值**：说话比打字更符合情感倾诉的心理习惯，大幅降低表达门槛

### 1.4 语音输出 (TTS)

- **行业对标**：豆包全语音交互、ChatGPT Advanced Voice
- **当前状态**：AI 只能"打字"，不能"说话"
- **方案**：接入浏览器原生 `SpeechSynthesis` API（免费，多语音选择）
- **关键价值**：声音是情感连接的核心维度，"听到"AI 说话的温度远高于"看到"文字

### 1.5 情绪感知/追踪

- **行业对标**：Character.AI 情绪识别、Replika 情绪追踪
- **当前状态**：AI 完全不感知用户情绪状态
- **方案**：定期分析对话内容提取情绪标签（开心/焦虑/悲伤/愤怒），注入 system prompt 引导 AI 调整回复风格
- **关键价值**：情感疗愈类应用的基础能力，感知情绪才能做出有温度的回应

### 1.6 主动问候/每日关怀

- **行业对标**：Replika 主动发消息、豆包"早安/晚安"推送
- **当前状态**：AI 完全被动，从不主动发起对话
- **方案**：打开应用时 AI 主动打招呼，记录上次对话时间，问候语根据时间/季节/上次情绪动态变化
- **关键价值**：主动性是陪伴感的关键特征，被动等待是工具不是伴侣

### 1.7 情绪日记 / 心情追踪

- **行业对标**：Daylio、Moodnotes、简单心理
- **当前状态**：无情绪记录功能
- **方案**：用户记录每日心情（😊😢😡😰），AI 追踪情绪趋势，在聊天中引用历史情绪数据
- **关键价值**：情绪疗愈的量化维度，给用户提供自我觉察的工具

## 二、对话体验

### 2.1 对话分支可视化

- **现状**：树形数据结构已完整实现（`showPreviousMessage`/`showNextMessage`），但 UI 未暴露 ← → 按钮
- **成本**：零——两个箭头按钮，5 行代码
- **方案**：在助手消息旁增加 ← → 箭头按钮，切换同一用户消息下的不同分支回复

### 2.2 消息编辑

- **现状**：消息发送后无法编辑，打错字只能重新发一条
- **方案**：用户消息旁加"编辑"按钮，编辑后自动重新生成 AI 回复
- **行业对标**：ChatGPT、Claude、Open WebUI 全部支持

### 2.3 消息删除（单条）

- **现状**：只能删除整个对话，不能删单条消息
- **方案**：每条消息旁加"删除"按钮，连带删除该消息及其子分支

### 2.4 打字指示器

- **现状**：流式生成时无任何视觉提示，用户不知道是否有回复在生成
- **方案**：AI 头像旁显示跳动的三个点动画或"正在输入..."文字

### 2.5 流式性能指标

- **现状**：`message.info` 有完整的 token/s、总耗时、加载耗时等数据，从未渲染
- **方案**：助手消息底部灰色小字显示 Token/s、生成耗时

### 2.6 自动滚动优化

- **现状**：用户向上翻阅历史时，流式内容强制拉回底部
- **方案**：手动滚动时暂停自动滚动，流式完成后显示"回到底部"浮动按钮

### 2.7 上下文长度提示

- **现状**：无法查看当前已用/总 token 数
- **方案**：输入框附近显示 token 估算计数器

### 2.8 错误重试

- **现状**：失败消息显示错误文本，无重试入口
- **方案**：失败消息旁加"重新发送"按钮

## 三、内容能力

### 3.1 文件/图片上传（多模态）

- **行业对标**：ChatGPT、Claude、Kimi 全部支持
- **方案**：支持粘贴/拖拽图片，转 base64 传给多模态模型（qwen3 等已支持）

### 3.2 Markdown 导出

- **行业对标**：ChatGPT 可复制为 Markdown
- **方案**：消息复制增加"复制为 Markdown"选项，或整段对话导出为 .md

### 3.3 URL 阅读

- **行业对标**：Kimi 粘贴 URL 自动抓取内容
- **方案**：检测输入中的 URL，自动抓取网页文本注入上下文

### 3.4 网页搜索

- **行业对标**：ChatGPT Search、Kimi 联网搜索、豆包联网
- **方案**：接入 Ollama tool calling 或 SearXNG 自建搜索

### 3.5 消息引用回复

- **行业对标**：微信引用回复、Slack threads、Open WebUI 引用
- **方案**：长对话中可引用之前某条消息再回复，提高上下文连贯性

## 四、管理与组织

### 4.1 对话置顶/归档

- **方案**：每条对话加 Pin 和 Archive 标记，侧边栏分区显示（置顶/活跃/归档）

### 4.2 提示词库

- **现状**：无预设提示词模板
- **方案**：内置心理疏导、情绪安抚、日常聊天等提示词模板，支持用户自定义保存

### 4.3 对话标签/分组

- **方案**：支持用户创建自定义标签（如"工作压力""亲密关系""自我成长"），按标签筛选对话

### 4.4 对话分享

- **方案**：生成只读分享链接或导出为精美长图/卡片

### 4.5 数据统计面板

- **方案**：显示对话次数、AI 调用次数、token 消耗、使用时长等统计

### 4.6 隐私对话模式

- **行业对标**：ChatGPT Temporary Chat、豆包"悄悄话"
- **方案**：一键开启不保存的私密对话，降低倾诉心理门槛
- **关键价值**：情感倾诉涉及隐私敏感话题，不保存的承诺是建立信任的基础

## 五、平台能力

### 5.1 PWA 离线支持

- **现状**：已有 `manifest.json` 但无 Service Worker
- **方案**：注册 Service Worker，缓存静态资源，支持离线打开和桌面快捷方式

### 5.2 桌面通知

- **方案**：AI 回复完成时发送桌面通知（用户可开关）

### 5.3 键盘快捷键

- **方案**：
  - `Ctrl+N` 新建对话
  - `Ctrl+/` 搜索对话
  - `Ctrl+Enter` 发送
  - `Esc` 关闭弹窗/侧边栏

### 5.4 响应式优化（移动端）

- **方案**：移动端全屏对话视图 + 底部 TabBar 导航，输入框适配虚拟键盘

### 5.5 主题跟随系统

- **方案**：增加"跟随系统"选项，监听 `prefers-color-scheme` 自动切换深色/浅色

### 5.6 字体大小调节

- **方案**：在设置中增加字体大小滑块，适配不同视力需求的用户
- **关键价值**：无障碍基础功能，特别是面向可能有视觉疲劳的情绪低落用户

### 5.7 多语言统一

- **方案**：统一为中文，或引入 i18n 框架支持多语言切换

## 六、安全与运维

### 6.1 环境变量配置

- **现状**：数据库密码、JWT 密钥硬编码在源码中
- **方案**：迁移到 `.env` 文件，通过 `process.env` 读取

### 6.2 密码重置流程

- **现状**：无"忘记密码"功能
- **方案**：增加密码重置 API（邮箱验证码）

### 6.3 速率限制

- **现状**：API 端点无任何限流
- **方案**：登录接口增加失败次数限制防止暴力破解；聊天接口增加频率限制

### 6.4 安全词/危机干预

- **行业对标**：Character.AI 自杀预防、Replika 危机热线
- **现状**：无任何安全检测机制
- **方案**：检测对话中的自伤/自杀关键词，触发安全提示并展示心理援助热线
- **关键价值**：情感疗愈类应用的社会责任，不能忽视
